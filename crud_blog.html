01 — CRUD
CRUD란 무엇인가

CRUD는 데이터베이스에서 할 수 있는 4가지 기본 연산의 앞글자를 딴 용어입니다. 세상의 모든 애플리케이션은 결국 이 네 가지 조합입니다. 병원 EMR도, 인스타그램도, 카카오톡도 CRUD입니다.

Create
INSERT
새 데이터 생성
환자 신규 등록
R
Read

데이터 조회
환자 목록 확인

데이터 수정
처방 변경 기록

Delete
DELETE
데이터 삭제
퇴원 후 처리

병원 EMR로 보면: 환자 입원 등록(C) → 차팅 조회(R) → 처방 변경(U) → 퇴원 기록 삭제(D). 매일 하는 업무가 전부 CRUD입니다.

SQL — CRUD 한눈에 
-- C: 생성
INSERT INTO patients (name, age, dx) VALUES ('홍길동', 65, 'DM');

-- R: 조회
SELECT * FROM patients WHERE age >= 60;

-- U: 수정
UPDATE patients SET diet = '당뇨식' WHERE name = '홍길동';

-- D: 삭제
DELETE FROM patients WHERE id = 1;
02 — CREATE
CREATE란 무엇인가

CREATE는 테이블의 구조(틀)를 만드는 명령어입니다. INSERT로 데이터를 넣기 전에 반드시 CREATE로 테이블을 먼저 만들어야 합니다. 엑셀로 치면 시트를 만들고 열 이름을 정하는 것입니다.

기본 문법
SQL — CREATE TABLE 
CREATE TABLE patients (
    id          INT           PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(50)   NOT NULL,
    age         INT,
    dx          VARCHAR(100),
    diet        VARCHAR(50)   DEFAULT '일반식',
    created_at  DATETIME      DEFAULT NOW()
);

-- 테이블이 이미 있으면 에러 방지
CREATE TABLE IF NOT EXISTS patients ( ... );
주요 데이터 타입
INT
정수값. 나이, ID, 개수 등에 사용합니다.
예: age INT → 65, 42, 78
VARCHAR(n)
최대 n자까지 저장하는 가변 길이 문자열. 실제 입력된 길이만큼만 공간 사용합니다.
예: name VARCHAR(50) → '홍길동'
TEXT
긴 텍스트. 차팅 내용, 메모처럼 길이 예측이 어려운 내용에 사용합니다.
예: notes TEXT → 긴 간호 기록
DATETIME
날짜와 시간 저장. 입원일, 처방 시각 등에 사용합니다.
예: 2025-01-15 14:30:00
BOOLEAN
참/거짓. 퇴원 여부, 활성 상태 등 이진 값에 사용합니다.
예: is_discharged BOOLEAN → true/false
DECIMAL(p, s)
소수점이 필요한 정밀한 숫자. 약품 용량, 체중 등에 사용합니다.
예: weight DECIMAL(5,2) → 72.50
제약조건 (Constraints)

제약조건은 데이터의 무결성을 보장하는 규칙입니다. 잘못된 데이터가 들어오는 것을 DB 레벨에서 막아줍니다.

  • PRIMARY KEY각 행을 유일하게 구별하는 기본키. NOT NULL + UNIQUE가 자동 적용됩니다.
  • AUTO_INCREMENT새 행이 삽입될 때 자동으로 1씩 증가. id 컬럼에 주로 사용합니다.
  • NOT NULLNULL 값을 허용하지 않습니다. 반드시 값을 입력해야 하는 컬럼에 사용합니다.
  • DEFAULT 값값을 입력하지 않으면 지정한 기본값이 자동으로 들어갑니다.
  • UNIQUE중복 값을 허용하지 않습니다. 이메일, 면허번호 등에 사용합니다.
  • FOREIGN KEY다른 테이블의 PK를 참조하여 테이블 간 관계를 설정합니다.
SQL — 실제 병원 테이블 예시 
CREATE TABLE prescriptions (
    id          INT           PRIMARY KEY AUTO_INCREMENT,
    patient_id  INT           NOT NULL,
    doctor_id   INT           NOT NULL,
    medication  VARCHAR(100)  NOT NULL,
    dosage      VARCHAR(50),
    issued_at   DATETIME      DEFAULT NOW(),

    -- FK: 다른 테이블과 관계 설정
    FOREIGN KEY (patient_id) REFERENCES patients(id),
    FOREIGN KEY (doctor_id)  REFERENCES doctors(id)
);

ERD → CREATE 순서: ERD로 테이블 구조를 설계한 뒤, CREATE TABLE로 실제 DB에 구현합니다. FK가 있는 테이블은 참조하는 테이블(patients, doctors)을 먼저 CREATE해야 합니다.

03 — INSERT
INSERT란 무엇인가

INSERT는 CREATE로 만든 테이블에 실제 데이터를 삽입하는 명령어입니다. CRUD의 C(Create)에 해당합니다.

기본 문법
SQL — INSERT INTO 
-- 기본 INSERT: 컬럼 명시
INSERT INTO patients (name, age, dx, diet)
VALUES ('홍길동', 65, 'DM, HTN', '당뇨식');

-- id, created_at은 AUTO_INCREMENT / DEFAULT로 자동 처리

-- 여러 행 한 번에 INSERT
INSERT INTO patients (name, age, dx, diet)
VALUES
    ('이영희', 42, 'HTN',  '저염식'),
    ('김철수', 78, 'DM',   '당뇨식'),
    ('박민준', 55, '일반', '일반식');

-- 전체 컬럼 순서대로 넣을 때는 컬럼명 생략 가능 (비권장)
INSERT INTO patients
VALUES (DEFAULT, '최수연', 61, 'DM', '당뇨식', DEFAULT);

컬럼명을 명시하는 게 좋은 이유: 컬럼 순서가 바뀌거나 새 컬럼이 추가돼도 기존 INSERT가 깨지지 않습니다. 실무에서는 항상 컬럼명을 명시합니다.

주의: INSERT 시 NOT NULL 컬럼에 값을 넣지 않으면 에러가 발생합니다. name 컬럼이 NOT NULL이라면 반드시 값을 입력해야 합니다.

SQL — INSERT 에러 상황 
-- ✗ NOT NULL 컬럼에 NULL 삽입 → 에러
INSERT INTO patients (name, age)
VALUES (NULL, 65);
-- ERROR: Column 'name' cannot be null

-- ✗ FK 참조 대상이 없을 때 → 에러
INSERT INTO prescriptions (patient_id, medication)
VALUES (9999, 'Amoxicillin');
-- ERROR: Cannot add or update a child row: foreign key constraint fails

-- ✓ 올바른 INSERT
INSERT INTO patients (name, age, dx)
VALUES ('최수연', 61, 'DM');
-- Query OK, 1 row affected
04 — 실습
실습 코드

아래 순서대로 직접 실행해보세요. 인수인계 시스템의 patients 테이블을 처음부터 만들어봅니다.

실습 순서
  1. CREATE TABLE로 patients 테이블 구조 생성
  2. INSERT로 환자 5명 데이터 삽입
  3. SELECT *로 삽입된 데이터 확인
  4. 컬럼명 생략 INSERT와 명시 INSERT 차이 직접 확인
  5. NOT NULL 위반 INSERT를 일부러 시도해서 에러 메시지 확인
SQL — 전체 실습 코드 
-- STEP 1: 테이블 생성
CREATE TABLE IF NOT EXISTS patients (
    id          INT          PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(50)  NOT NULL,
    age         INT,
    dx          VARCHAR(100),
    diet        VARCHAR(50)  DEFAULT '일반식',
    created_at  DATETIME     DEFAULT NOW()
);

-- STEP 2: 환자 데이터 삽입
INSERT INTO patients (name, age, dx, diet)
VALUES
    ('홍길동', 65, 'DM, HTN', '당뇨식'),
    ('이영희', 42, 'HTN',     '저염식'),
    ('김철수', 78, 'DM',      '당뇨식'),
    ('박민준', 55, '일반',    '일반식'),
    ('최수연', 61, 'DM',      '당뇨식');

-- STEP 3: 확인
SELECT * FROM patients;

-- STEP 4: diet를 입력하지 않으면 DEFAULT '일반식'이 들어감
INSERT INTO patients (name, age, dx)
VALUES ('테스트', 30, '일반');

SELECT * FROM patients WHERE name = '테스트';
-- diet 컬럼이 '일반식'으로 자동 입력됨을 확인

-- STEP 5: NOT NULL 위반 (에러 확인용)
INSERT INTO patients (name, age)
VALUES (NULL, 30);
-- ERROR: Column 'name' cannot be null
